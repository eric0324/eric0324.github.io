利用 GitHub 建立一個 Blog

Posted on 2016-04-27 | Edited on 2018-10-22 |

我一直很想架設一個個人網站很久了，想把之前的文章、筆記、Hackpad 、心得做個整理。不過事情太多，一直拖到現在才用，現在有這麼多的免費服務和工具，沒有理由再拖延下去了。

這一切的設置都只要 30 分鐘就可以搞定，完成一個乾淨看起來有質感的個人網頁，我非常滿意，所以記錄下來，希望可以幫助更多的人創建自己的個人網站。

首先 jekyll 是基於 ruby 的，所以要先安裝 ruby（可以參考我之前的教學）。

接著使用 Command Line 安裝 Jekyll

新增一個 Jekyll 專案並使用 jekyll new來初始化。完成之後使用 jekyll build和 jekyll serve來建置一個位於本地端的部落格。

詳細的流程可以參考 Jekyll Quick-start 頁面。

Github

首先你必須要先有個 Github 帳號，建立一個 repository ，如此一來你就會有個 Github Pages 。

特別注意的是，在命名 repository 時有兩種選擇：

  • 自訂專案名稱
  • 命名為 你的帳號.github.io

使用自訂專案名稱的話，你的 base URL 會是 你的帳號.github.io/你的專案名稱，同時必須將專案內容放到 gh-pages 的分支下，才能夠正常建置。每一個 Github 帳號能夠擁有多個專案網站。

另一方面，專案使用特定名稱 你的帳號.github.io 的話，此儲存庫的 Github Page 會成為使用者網站。部落格的 base URL 則是 你的帳號.github.io/。

確定加入所有內容並 Push 後就算完成第一步。

結構

接著讓我們來看看一些基本架構（我只挑我想講的講）：

_posts 資料夾是用來存放文章
新增文章的方法很簡單，只需要丟一份 markdown 檔案到該資料夾內，Jekyll 就會自動生成新頁面了。檔案命名規則是 年-月-日-文章標題.md。

_config.yml 是很重要的檔案，負責部落格的設定管理，可設定的內容包含網址路徑，網站描述，markdown 的寫法，你想得到的都可以設定。

還有一個 about.md 則是自我介紹頁面，你可以在裡面透過 markdown 撰寫自我介紹。
