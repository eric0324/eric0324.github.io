MiKA 咪卡

Posted on 2019-01-01 | Edited on 2019-04-21 |

GoodLinker 這間公司以邊緣運算為基礎，開發各式各樣的應用，正因為認同創辦人理念，所以加入這個新創團隊。其中這個題目主要是利用邊緣運算去做影像分析，使用者可以透過相機拍攝名片後傳給 chatbot ，得到照片後開始分析資訊回傳給 chatbot ，讓使用者更方便的建立名片。

MiKA服務介紹影片

情境觀察與痛點分析

台灣人對於 LINE 有很強大的依賴性，因此很多社交行為也會在 LINE 上面發生，GoodLinker 觀察到很多商務人士在 LINE 上做名片的交換。不過這交換過程，我們發現很多問題：

  1. 名片拍攝的相當模糊。
  2. 名片有正反面，所以要存兩張照片。
  3. 收到名片後還要『儲存圖片』，相當麻煩。

服務/產品設計

透過 LINE Chatbot 技術、OCR 光學字元辨識技術，讓使用者可以直接對著我們 Chatbot 傳送照片快速建立數位化名片，加快人與人之間的連結。

智能名片

使用者可以直接透過相機拍攝名片後傳送給 MiKA，MiKA 會透過 OCR 技術去分析名片上的內容，解析文字後協助使用者建立電子名片

客製化名片

有別於傳統名片制式化設計，每張名片在傳送的時候都是透過 LINE chatbot 的 flex message 去建立，Hero 的 image 都會放上我們自行設計的角色圖，點選角色圖後還有會遞出名片的動畫，而這些名片和動畫都可以透過 MiKA STORE 去做選購和更換

群組活動

名片建立後，更多商務人士會需要群組上的互動，那群組活動將會是很重要的一環，因此又開發了群組活動功能，而這個群組活動功能有別於 LINE 原生的活動功能，活動還能公開在 MiKA 上被搜尋到，增加活動曝光機會。

核心技術

OCR 名片分析

OCR 的運算上相當的耗能，GoodLinker 的開發重點在於邊緣運算技術，邊緣運算（Edge computing），這是一種分散式運算的架構，透過邊緣運算將應用程式、數據資料與服務的運算，由網路中心節點，移往網路邏輯上的邊緣節點來去做處理，如此一來可以降低雲端運算上的負擔

MiKA STORE

我們設計了一系列的遞出名片的動畫和名片角色，讓使用者可以更加客製化自己的名片，擺脫一成不變的名片。

面對挑戰

這個專案的挑戰在於 UX 和對於用戶的名片該如何做解析。

就 UX 而言，分析名片這件事情是需要 Machine Learning 的，但是使用者無法理解這件事，所以會給予很高的期待，剛推出的時候常常會有使用者抱怨名片都分析不出來，因此在這裡做了很多流程和文字上的著墨。接著，使用者如果分析失敗，會一直傳送照片作分析，這對於後端的伺服器會有一定的負擔在

再來是圖片分析的問題，很多使用者在拍攝的時候常常會角度傾斜，導致名片效果變差，於是我試著將每張圖進入 OCR API 的時候進行圖片的矯正

原先預期這會是個好方法，可以協助 OCR 的分析更加精準，沒想到這樣翻轉後，連文字也一起被翻轉，反而降低 OCR 翻譯的命中率，因此只好先回到原先的作法，不要翻轉直接做掃描分析
